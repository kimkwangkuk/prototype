# Todo App - 프로토타입 프로젝트

## 프로젝트 철학

**이 프로젝트는 프로덕션이 아닌 프로토타입입니다.**

### 목적
- 다양한 UI/UX 패턴을 **빠르게 실험하고 비교**
- 실제 서비스가 아닌 **아이디어 검증용 프로토타입**
- 완벽함보다 **속도와 유연성** 우선

### 개발 원칙
- ⚡ **빠른 구현**: 복잡한 아키텍처보다 동작하는 코드
- 💰 **경제적 개발**: 최소한의 의존성, 빌드 시간 최소화
- 🔄 **빠른 반복**: 쉽게 만들고, 쉽게 버리고, 쉽게 다시 만들기
- 🎯 **현재 요구사항**: 확장성보다 지금 필요한 기능에 집중

---

## 핵심 패턴: Variants

**같은 기능의 여러 UI 버전을 한 번에 실험**

```javascript
// config.js에서 한 줄만 바꾸면 전체 UI 변경
export const CURRENT_VARIANT = 'b'; // 'a', 'b', 'c' 중 선택

// 각 컴포넌트는 variant별 구현 제공
const variants = {
  a: () => renderComponent_A(),
  b: () => renderComponent_B(),
  c: () => renderComponent_C(),
};
```

### 현재 Variants

| Variant | 설명 | 특징 |
|---------|------|------|
| **A-type** | 기본 투두리스트 | 인라인 편집, 전통적 UI |
| **B-type** | 실험적 UI | 바텀시트 중심, 인라인 편집 없음 |
| **C-type** | 예정 | 새로운 실험용 |

---

## B-type 핵심 규칙

**현재 활성 variant의 주요 동작 방식**

### 1. 바텀시트 중심
- ✅ 모든 편집은 바텀시트를 통해서만
- ✅ 바텀시트 상단에 텍스트 입력 필드
- ❌ 인라인 편집 절대 금지

### 2. 실시간 반영
- 텍스트 입력 → 즉시 목록에 반영
- 카테고리 변경 → 즉시 목록에 반영

### 3. 취소 처리
- 취소 시 → 모든 변경사항 원본으로 복원
- 빈 텍스트 → 할일 삭제

## 기술 스택

**프로토타입에 최적화된 심플한 구성**

- **Vanilla JavaScript** (ES6+) - 프레임워크 없음
- **CSS Variables** - 동적 스타일링
- **Vite** - 최소한의 빌드 도구

### 금지 사항
- ❌ React, Vue 등 프레임워크
- ❌ TypeScript
- ❌ 복잡한 빌드 설정
- ❌ 이모지 (사용자 명시 요청 시만)

**이유**: 빠른 실험을 위해 설정과 빌드 시간을 최소화
